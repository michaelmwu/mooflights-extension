# Data Imports

The extension should not hand-edit generated reference data. Use local scripts to produce deterministic snapshots, then
review the diff before committing.

## Ownership Split

- Extension repo: reproducible import scripts, compact public fallback snapshots, fixture checks, and generated reports.
- Mu Travel backend: canonical refresh jobs, richer private metadata, reliability scoring, affiliate/direct-link rules, and
  account-specific preference data.

The extension must keep working without the backend, but it should not bundle every private or brittle commercial rule.

## Current Scripts

- `bun run import:airports`: rebuild the compact canonical airport snapshot.
- `bun run import:airport-coordinates`: compatibility alias for `import:airports`.

Planned import scripts for mileage earnings, airline names, and generated-data validation should be added before those
snapshots become regularly refreshed.

## Mileage Earning Sources

Do not copy Where to Credit page contents into the extension. Where to Credit can remain an outbound lookup target, but
bundled earning-rate snapshots should be generated from airline/program public earning charts, licensed datasets, or
curated Mu Travel reference data.

## Airline Code Sources

Candidate sources:

- ITA Matrix itinerary JSON: best runtime source for the current itinerary carrier display name.
- IATA current airline members: preferred source for active airline code/name data.
- Wikipedia list of airline codes: supplemental source. Keep only rows with valid two-character IATA codes and airline
  names that link to real airline article pages, not index/list/disambiguation-style pages.

Generated airline snapshots should include source attribution and a `fetched_at` timestamp.

## Airports

`src/shared/data/airports.json` is generated by:

```sh
bun run import:airports
```

The script imports `airports.csv` from `https://davidmegginson.github.io/ourairports-data/airports.csv`, keeps only
medium and large airports with valid three-character IATA codes, and stores compact stable airport facts:

- airport code
- airport name
- city/municipality
- ISO country code
- continent name
- latitude and longitude as integer degrees multiplied by `10000`

OurAirports publishes this data into the public domain. Keep attribution to `https://ourairports.com/data/` in docs and
release notices when bundling this snapshot. The extension uses it for offline great-circle distance estimates and
search-page airport helper filtering; backend services may use a different source if Mu Travel needs closed-source or
proprietary enrichment.

`src/shared/data/airport-region-presets.json` is a curated local helper for inserting useful groups of airports into ITA
Matrix search fields. It should stay small and contain only stable airport-code groupings, not route service, alliance,
or airline-network claims.
